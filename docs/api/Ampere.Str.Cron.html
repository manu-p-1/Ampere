Namespace Ampere.Str.Cron

Classes

CronBuilder

A fluent, type-safe builder for constructing CronExpression instances.

Provides both raw field methods (WithMinutes(string)) and high-level convenience methods (EveryNMinutes(int), AtTime(int, int), OnDaysOfWeek(params DayOfWeek[])). 

var expr = new CronBuilder()
    .AtTime(9, 0)
    .OnDaysOfWeek(DayOfWeek.Monday, DayOfWeek.Wednesday, DayOfWeek.Friday)
    .Build();
// produces "0 9 * * 1,3,5"
CronExpression

An immutable, thread-safe representation of a fully parsed cron expression.

Supports standard 5-field (minute–DOW), 6-field with seconds, and 7-field with seconds + year formats. Also supports Quartz-style extensions: L, W, LW, #, and ?.

Use Parse(string) to obtain instances, or CronBuilder to construct them fluently.

CronField

Represents a single parsed field (e.g. minutes, hours) of a cron expression. A field contains one or more CronNode elements (the comma-separated parts). Instances are immutable and thread-safe.

CronHashNode

Nth day-of-week in the month (5#3 = third Friday).

CronLastDayNode

Last day indicator (L) for day-of-month, optionally with an offset (L-3).

CronLastWeekdayNode

Last weekday (LW) of the month – shorthand for the last business day.

CronLastWeekdayOfMonthNode

Last specific weekday of the month (5L = last Friday). Used in the day-of-week field.

CronNearestWeekdayNode

Nearest weekday to a given day-of-month (15W).

CronNode

Abstract base for all nodes in a cron-field AST.

CronParseException

Exception thrown when a cron expression cannot be parsed or validated. Provides detailed information about the parse failure including position and field context.

CronParser

A high-performance, enterprise-grade cron expression parser supporting standard 5-field (minute–DOW), 6-field (with seconds), and 7-field (with year) formats, plus Quartz-style extensions (L, W, LW, #, ?) and predefined macros (@yearly, @monthly, etc.).

All public methods are thread-safe. Parsed CronExpression instances are immutable and may be cached freely.

CronQuestionMarkNode

Question mark (?) – "no specific value", used in day-of-month / day-of-week.

CronRangeNode

A contiguous range (e.g. 1-5).

CronStepNode

A step expression (e.g. * /5, 1-10/2, 0/15). The Base is the range or wildcard; the Step is the increment.

CronValueNode

A single integer value (e.g. 5).

CronWildcardNode

Wildcard (*) – matches every value in the field's range.
